## 候補アイデア

### 1. あなたのアイデアを中立的に評価しよう
アイデアを入力 → AIが良い点・懸念点・実現可能性スコアを中立評価
- Streamlit + Claude API (haiku)
- ストリーミングでリアルタイム表示
- プログレスバーで実現可能性を視覚化

### 2. 現場あるある翻訳機
エンジニアの愚痴/モヤモヤを入力 → 3パターンで変換
- ポジティブ変換
- マネージャー報告文
- 退職届風

### 3. LT審査AI
LTタイトル＋あらすじを入れると採択確率・改善案・バズりタイトル案を出す
（LTイベントでやるからメタで面白い）

### 4. エンジニア市場価値診断AI ※イベントルールに抵触する可能性あり
スキル・経験年数を入力 → 推定年収レンジ・伸ばすべきスキルを出力
→ 採用・転職勧誘禁止ルールに引っかかるおそれがあるため保留

## 新規追加アイデア

### 5. Git blame 言い訳ジェネレーター 【ユーモア×技術系】
コミットログ＋diff を貼ると「当時の自分が書いた理由」を尤もらしく生成する
- 例: `// TODO: fix later` → 「当時は締め切り前夜でして……」
- 技術的負債あるある と絡めるとLTのオチになる
- 実装: diff テキスト → Claude に "当時の自分になりきって言い訳して" プロンプト

### 6. スタンドアップMTG盛り増しAI 【実用ツール系】
「昨日: PR出した / 今日: レビュー待ち / ブロッカー: なし」の1行入力を
マネージャーウケする3倍盛りのスタンドアップ原稿に膨らませる
- 実用性が高く、笑いと共感を両立できる
- 逆向きデモ（長い → 1行に圧縮）もセットで見せると面白い

### 7. 技術ブログ「バズりタイトル」リライター 【コンテンツ戦略系】
Zenn/Qiita の記事タイトルを入力 → バズり確率スコア＋改善案3パターンを出力
- 「〜してみた」「〜で詰まった話」パターン分析
- SEO的観点とエモーショナル観点の両軸評価
- 情報発信 = 市場価値向上 というイベントテーマとも直結

### 8. PRレビューコメント感情温度計 【職場あるある系】
GitHubのPRコメントを貼ると「このレビュアーの感情温度」を可視化
- 例: 「なるほど、ただ…」→ 🌡️45℃（やや不満）
- 例: 「LGTM」だけ → 🌡️20℃（興味なし）
- 観客の実際のPRコメントを会場でリアルタイム診断するデモが映える

### 9. 履歴書コードレビューAI 【逆転の発想系】
職務経歴書のテキストをコードに見立てて「コードレビュー風」にフィードバック
- 「この関数(職歴)、責任範囲が広すぎます。単一責任原則に反しています」
- 「命名規則が不明確です。"〇〇担当"は具体的にどんな処理ですか」
- エンジニアの市場価値文脈ともマッチ・笑いながら自己分析できる

### 10. 「それって要は」エンジニア語辞典 【文化人類学的アプローチ】
エンジニア組織特有の言い回しを収集・解析し、本音を対訳する
- 「リファクタリングしたい」→「このコード読みたくない」
- 「スケーラビリティが…」→「今は面倒なのでやりたくない」
- 観客に投票・追加してもらうインタラクションを入れると会場が盛り上がる

### 11. 技術選定ポエム自動生成AI 【風刺系】
「なぜXXXを採用したか」の技術ブログ書き出し部分を自動生成
- どんなスタックを入れても「〜の課題を感じていました。そこで我々は…」で始まる
- 技術選定の意思決定の曖昧さへの風刺として機能する
- 5分で実装できる最小デモ向き

### 12. エンジニア「詰まり時間」コーチAI 【内省・成長系】
今日ハマったエラーや検索クエリを貼ると「詰まりパターン分析＋改善提案」を出力
- 「あなたの今日の詰まりパターン: npm系 40% / 型エラー 35%」
- 「型定義を先に書くと35%の詰まりが解消する可能性があります」
- 市場価値向上 = 詰まり時間の削減 という切り口でテーマと接続できる

## Failure系アイデア（第2フェーズ）

### F1. Failure Digest AI
Gitコミット、Issue、PRから「今週の失敗と学び」を自動要約する。
- 失敗を単発イベントで終わらせず、再利用可能な知見に変える
- 実装: GitHub API + Claude で週次サマリー生成

### F2. Stuck Pattern Miner
エラー履歴・検索クエリから「自分が何で詰まりやすいか」を自動抽出する。
- 「あなたは型エラー型」「環境構築型」という分類で見せる
- 挑戦度高: 分類軸の設計と信頼性が肝

### F3. Retry Replay ⭐️最終推奨
失敗タスクの「最初の仮説→間違った試行→突破口」を時系列で自動再構成する。
- 結果だけでなく思考過程を資産化できる
- LT構成とアイデアの構造が一致（発表自体がRetry Replayになる）
- 実装: gitログ + コミットメッセージ → Claude で時系列再構成
- 発表スタイル「構成・ストーリー型」に最もフィット

### F4. Postmortem Copilot
障害やバグ対応のログから、ポストモーテムのドラフトを自動生成する。
- 実務導入しやすい。入力ソースを絞れば現実的
- LT映えは弱いが実プロダクト価値は高い

### F5. Missed Signal Detector
「本当は事前に気づけたサイン」をPRコメント・ログ・会話から後から抽出する。
- 「このコメントが予兆でした」という見せ方が刺さる
- 後知恵バイアスを自己言及的に語ることでLT映え可

### F6. Debug Journal AutoWriter
ターミナル履歴・コミット・ブラウザ検索語から、デバッグ日誌を自動生成する。
- ビフォーアフターが明快。デモ前提ならLT向け安定度トップ
- 思想寄り発表ではF3/F5に劣る

### F7. Team Failure Memory
チーム内の失敗事例を匿名化・構造化して、似た状況で再提示する共有知識ベース。
- 思想が最も強い。プロダクト化価値が高い
- LTでのデモは困難。プロダクト寄りで検討

### F8. Decision Regret Analyzer
技術選定や実装判断の記録から「後悔しやすい判断パターン」を出す。
- 失敗を意思決定の癖として扱う視点が新鮮
- 入力データの収集自体が難所

### F9. PR Lessons Extractor
PR上の指摘・修正・差し戻し履歴から「レビューで学んだこと」を自動抽出する。
- GitHub API連携で入り口が明確
- デモ前提ならF3と並ぶ最有力候補

### F10. Failure-to-Portfolio AI
失敗体験を、職務経歴書や面接で使える「改善・再発防止・学び」の表現に変換する。
- イベントテーマ「市場価値」との接続が最も直接的
- 他アイデアのアウトプット層として組み込むと強い

## 追加アイデアメモ

- AIにLTタイトルを100個考えさせたら、9割つまらなかった話
- 5分LTはAIより“雑な違和感”の方が強い
- Claudeに大阪のLTでウケる話を考えさせたら、たぶん東京っぽくなった
- AIに“現場感”を入れさせるのが一番むずかしい
- LTネタをAIに出させて、人間がツッコむゲームを作る
- AIに自分の市場価値を説明させたら、盛りすぎた
- 転職っぽい自己紹介をAIに書かせたら、信用できなくなった
- AIが考えたポートフォリオは強そうに見えて弱い
- “市場価値”を点数化するAIを作ったら嫌な気持ちになった
- SES、自社開発、フリーランスをAIに選ばせたら全部おすすめしてきた
- 現場で使えるAIと、LTで映えるAIは別物だった
- AIに“エンジニアのリアル”を語らせたら解像度が足りなかった
- 5分で作ったAIサービス、5分で飽きた
- AIと一緒にLTを作ると、自分の雑さが武器になる
- AIにキャリア相談したら、だいたい正しいけど何も刺さらない
- “それっぽい”を量産するAIに、どう勝つか
- AIに自己分析させたら、自己肯定感が高すぎた
- AIでLTを自動生成したら、最後に必要なのは人間の悪ノリだった
- 市場価値を上げたい人向けのAIを作ったら、まず自分が刺された
- AIに“今のままでいいのか”を聞いたら、急に重くなった
